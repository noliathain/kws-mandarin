# Architecture

```
waveform (16 kHz mono)
   │
   ├─ (train only) MUSAN + RIR augmentation across SNRs down to 0 dB
   │
log-mel front-end            40 mel bins, 25 ms / 10 ms
   │
BC-ResNet encoder            broadcasted residual blocks + SubSpectralNorm, scale 3–6
   │                         streaming / bounded-lookahead (causal convs)
   │
CTC head                     softmax over tonal-pinyin units (+ blank)
   │
   ├─ training:  CTC loss, optionally with NTC-style wildcard arcs
   │             (self-loop arc = noise insertion, bypass arc = masking)
   │
   └─ inference: open-vocabulary keyword search over the posterior lattice
                 plain CTC now │ MFA-KWS TDT head later (same interface)
```

## Units (tokenizer)

A keyword is a runtime token sequence, so the inventory must be fixed, complete, and
compositional. We derive it from tonal pinyin at one of three granularities:

| Mode       | Example `中` | Inventory | Trade-off |
|------------|-------------|-----------|-----------|
| `FINAL` (default) | `zh` + `ong1` | ~300 | tone stays on its final; good data efficiency |
| `SEPARATE` | `zh` + `ong` + `1` | ~85 | smallest; tone detached from final (less natural) |
| `SYLLABLE` | `zhong1` | ~1700 | matches CRNN-CTC precedent; large softmax |

Splitting a TONE3 syllable into initial/final/tone is a pure string operation, so
pypinyin's phrase-level heteronym (多音字) disambiguation is preserved. The inventory is
generated by [../scripts/build_vocab.py](../scripts/build_vocab.py) and committed under
`src/kws_mandarin/tokenizer/vocab/`. Id 0 is the CTC blank; id 1 is `<unk>`.

## Encoder

BC-ResNet. Two export-critical constraints, designed in from the start rather than
retrofitted:

1. **Streaming / causal.** Always-on KWS needs bounded latency, so convolutions are causal
   or use a small fixed lookahead. Vanilla BC-ResNet is not causal — this is a build
   requirement, not an afterthought.
2. **Quantization-friendly.** SubSpectralNorm's per-subband affine folds into the preceding
   conv at export; keep activations int8-friendly for a future PT2E path.

## Decoder (open-vocabulary)

Stable interface: `score(posteriors, keyword_units) -> detection_score` plus a streaming
variant. Two implementations behind it:

- **Plain CTC keyword search** (first): prefix/keyword scoring over the CTC posterior.
- **MFA-KWS TDT** (later): CTC + Token-and-Duration Transducer for better duration modeling;
  reference streaming search in X-LANCE/KWStreamingSearch.

## Robustness stack

- **NTC-style wildcard arcs** in CTC training/decoding — the highest-leverage single change
  for low-SNR false-alarm suppression.
- **Multi-SNR augmentation** — MUSAN (noise/music/babble) + RIR reverberation, on the fly,
  down to 0 dB SNR.
- **Confusable hard negatives** — LLM-generated tone-confusable strings (Tone 2/3, neutral)
  to suppress false accepts on near-homophones.

## Evaluation
